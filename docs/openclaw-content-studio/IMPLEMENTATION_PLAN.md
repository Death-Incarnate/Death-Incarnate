# Implementation Plan

## Phase 1 — Foundations

- Monorepo layout (`apps`, `packages`, `workers`).
- Postgres + Redis + object storage.
- Drizzle migrations + Zod contracts.

## Phase 2 — Core Content Domain

- Template CRUD
- Content job lifecycle
- Asset library + tagging
- Export record tracking

## Phase 3 — Agent Orchestration

- OpenClaw orchestrator with queue bindings
- Trend agent + script agent + scene agent
- Retry and dead-letter handling

## Phase 4 — AI Asset Layer

- Nano Banana 2 adapter
- TTS adapter
- Caption timing utility
- Asset normalization pipeline

## Phase 5 — Remotion Engine

- Template registry by ratio
- Composition builder from scene contracts
- Render worker and storage upload

## Phase 6 — Publishing

- YouTube adapter
- TikTok adapter
- Instagram adapter
- X adapter

## Phase 7 — Dashboard + Operations

- Jobs dashboard
- Batch generation UX
- Export history and failure insights
- Runbook and alert routing

## Phase 8 — Optimization

- Metric ingestion
- Prompt variant A/B testing
- Automatic strategy updates
